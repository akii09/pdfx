# Code of Conduct

PDFx is an open project. Everyone participating in it  through issues, PRs, discussions, or code review is expected to keep it that way.

## What we expect

Be respectful. Give feedback that targets the work, not the person. Assume good intent, especially from first-time contributors. If you make a mistake, own it and move on.

## What we don't accept

- Personal attacks, insults, or sustained hostile behavior
- Discriminatory or demeaning language of any kind
- Unwanted sexual language, imagery, or attention
- Intimidation, threats, or deliberate disruption
- Sharing someone's private information without their permission

## Scope

This applies everywhere the project exists: GitHub issues, pull requests, discussions, and any other official channel.

## Reporting

If something feels wrong, report it through [GitHub Discussions](https://github.com/akii09/pdfx/discussions). Include relevant context - links, screenshots, timeline - so it can be reviewed fairly.

## Enforcement

Maintainers may warn, temporarily restrict, or permanently remove someone depending on the severity and pattern of behavior. The goal is to keep the project safe and constructive, not to punish.
